Project Goal & Overview

This assignment required deconstructing and modifying a functional drag-and-drop script to create a stand-alone assessment tool. The goal was to transform the script's topic and modernize its visual design to demonstrate proficiency in both structural DOM manipulation and CSS responsiveness.

View Live Drag-n-Drop Site (Vercel)

Functional & Thematic Customization

The core logic was preserved while the application's content was completely overhauled:

  • New Topic: The original learning theory concepts were replaced with a Football Player Positions theme.
  • Assessment Structure: The application now features three primary position boxes: Forwards, Midfielders, and Defenders.
  • Draggable Items: Famous player names (e.g., Messi, ModriÄ‡, van Dijk) were implemented as draggable elements.
  • Constraint & Feedback: Each drop zone accepts a maximum of three items, and the application provides immediate color feedback (green for correct, red for incorrect) upon dropping, preserving the original assessment functionality.

Visual Redesign: CSS and Responsiveness

The accompanying CSS file was completely reworked to achieve a modern and appealing user interface:

  • Modern Aesthetics: I introduced a football-themed blue color palette, smoother borders, rounded boxes, and soft shadows to improve visual hierarchy.
  • Responsive Layout: The layout was redesigned using Flexbox to ensure all sections align properly and remain fully responsive across desktop, tablet, and mobile screen sizes. This was a critical step in moving away from the original fixed-width layout.
  • Consistency: Typography, padding, and margins were adjusted for a cleaner, more consistent interface and optimal readability.

Challenges, Troubleshooting, and Technical Growth

This project presented several valuable challenges related to deployment and responsive design:

  • Initial Alignment Issues: The original code used fixed widths, which caused spacing and alignment problems after introducing new content.
    Solution: I resolved this by switching the container styles to use flexible containers and percentage-based sizing to ensure fluid resizing.
  • Vercel 404 Error: The deployment initially failed with a 404 error because the main entry file was named `drag.html`.
    Solution: I fixed the deployment by renaming the entry file to `index.html`, which Vercel expects as the root file.
  • Maintaining Functionality: Ensuring that the drag-and-drop logic remained intact after structural and stylistic modifications required careful preservation of the original JavaScript event handlers.

Overall, this project significantly strengthened my understanding of DOM manipulation, advanced CSS responsiveness, Git version control, and the process of static site deployment using platforms like Vercel.
